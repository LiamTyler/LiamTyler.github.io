Seagate: Software Developer Internship (Summer 2016)

Documentation Generator

The team that I was a part of had their own specific coding style and conventions. It was different enough from standard styles where they couldn't really use doxygen or any other documentation generating tool, unless they wanted to go back and edit all of their comments and in-code documentation. They still wanted there to be one centralized place they could go to and see documentation for any of their projects though.

What I did was create a website that would do a 'git pull' on each of their projects, every 5 minutes. If there were any changes, for each file that needed documentation generated, it would call a custom parser that I had built. It would then generate an accordion style menu. You could navigate through it to any class or function. Once you clicked on a class or function, all of the documentation for that item would be neatly displayed.

This project was for the second half of my summer. Sadly I can't show the website or any of the code because it is Seagate's property.

Tools Used

  • Django for the website and storing the documentation
  • Pure CSS for accordion menu
  • Python for the custom parser

Features

  • Accordion menu
  • Self updating
  • Auto complete search bar for any function or class

Failure Analysis Website

This was a website for other teams inside of Seagate to make analysis requests to my group. This website had been built by an intern the previous semester, but it needed updating. There wasn't really any styling, a few things were broken, and they wanted more functionality. I set up a coulpe meetings where I got the requirements from a couple different teams, and then spent the first half of the summer implementing them. See below for descriptions of the features I updated/added.

Difficulties I had

I had never done any website programming before this. So, to suddenly have to edit Django, HTML, CSS, and Javascript code, as well as Ansible files, was a lot for me to get used to. I had a mentor who I definitely asked many many questions to, but most of my learning was just from thousands of Google searches and looking at past Stack Overflow answers. It took a while for me to get it all wrapped around my head, but by the end of the summer I was the expert on anything related to that website.

Tools Used

  • Django for the website
  • Ansible to update the server hosting the website

Portions I Updated

  • Added styling and color for the whole website
  • For the tables that holded requests, I added extra search features
  • Fixed a long standing bug where nobody could be added as an admin to the website
  • Organization in general. The project was a bit of a hurried mess by the time I got there

Portions I Added From Scratch

  • Added Javascript code ontop of the Datatables library to implement frozen columns and scrolling
  • Created essentially a forum for people to post technical tips that they had discovered in the lab. They used to have this with Lotus Notes, but they lost it when they switched away from Lotus Notes years ago
  • Created a tool to add large batches of people to the website without having to manually do a 5 step process for each person
