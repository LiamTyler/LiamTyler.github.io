Real Time Ray Tracing

Description

For the final project, I chose to implement a Cuda ray tracer that allowed a user to walk through a scene in real time. I uses bounding volume heirarchies (BVHs) as well as several other optimizations to enable rendering of scenes with millions of triangles in just a few milliseconds. This, combined with ability for objects to be both reflective and refractive, makes it a lot better than the last ray tracer I made.

Code

Code from this project can be found in this GitHub repo. The code is written in Cuda and C++ and it was built on top of my game engine Progression to help with things like windowing, input handling, and scene loading.

Features

  • Real-time 3D ray tracer
  • BVH traversal of triangle meshes
  • Optimized ray shape intersection tests
  • First person camera to move through the scene
  • Blinn phong shading
  • Skybox support
  • Limited support for dynamic objects
  • Tracing of spheres and triangle meshes
  • Scene loading from text files

Controls

  • WASD/Mouse: Camera control

Implementation Details

For more details on the performance and optimizations I did, take a look at the report I had to write on the project here.

Future work

I was really happy that the BVH enabled the scene to have millions of triangles in it, but one of the biggest limiting factors is that there is no BVH for the whole scene. Since the scene is dynamic, a new BVH would have to be created or updated every frame in just a few milliseconds. I did not have time to tackle this issue, but it would be the next major step. Another thing is that currently no anti-aliasing is done. This is fairly noticeable when looking at edges while the camera is moving, and on the very tiny triangles of the Buddha model.
